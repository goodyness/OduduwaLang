What happens when you merge ancient cultural heritage with modern software engineering? Over the past few weeks, I built **OduduwaLang**—a fully functional programming language written natively in Yoruba. 🇳🇬🌍

Language is how we shape the future, yet entire continents are often left behind in how we instruct machines. I wanted to see what it would look like to build a deep architectural system that honors one of West Africa’s most profound tongues.

**OduduwaLang v0.2.0** is now officially open-source and live on PyPI.

🛠️ **How it works under the hood:**
OduduwaLang uses a custom lexer capable of intelligently parsing structural Yoruba grammar (and diacritics). It builds an Abstract Syntax Tree (AST) and transpiles beautifully into Native Python. This means it has the full power of the vast Python ecosystem, completely abstracted behind native Yoruba.

✨ **Key Features:**
* **Object-Oriented Programming (OOP):** Complete class support using `egbe` (Class), `__ibere__` (Constructor), and `ara` (Self).
* **Native Exceptions:** Try/Catch blocks using `gbiyanju` (try), `mu_asise` (except), and `ni_ipari` (finally).
* **The `onka` Module:** A custom standard library that dynamically converts raw integers into spoken Yoruba vocabulary (e.g., `onka.yipada(1050)` compiles to “ẹgbẹ̀rún kan le àádọ́ta”).
* **Developer Ecosystem:** I built a fully functional CLI formatter (`oduduwa se_eto`), a Linter (`oduduwa yewo`), and even published an official VS Code Syntax Extension.

It started as an experiment in building compilers, but it evolved into something that proves technology doesn't have to exist in one language.

I would love for fellow software engineers, language enthusiasts, and open-source contributors to test it out, report bugs, or even help expand the standard library!
